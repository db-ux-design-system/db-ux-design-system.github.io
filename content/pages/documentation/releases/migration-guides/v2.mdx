## Rework Foundation

### Typography (Typo Tokens / Styles)

- **Unified font for code and design:** Replacing the previous fonts (DB Screen Sans in code / DB Screen Flex in Figma) with **DB Neo Screen Sans** and **DB Neo Head** in the provided typo tokens.
- **Italic added:** Introduction of an _italic_ style for the body token.
- ‼️ Note: the **DB Neo Screen Flex** font will be **removed from Figma by end of 2025**. Replace it in time to prevent future issues.

<img
	src="/assets/migration-guides/v2/typo.png"
	alt="Typography Overview"
	className="full-width-image"
/>

### Color Variables

- **New color class:** Introduction of **vibrant** for more vivid color usage.

<img
	src="/assets/migration-guides/v2/colors-vibrant.png"
	alt="Vibrant Colors"
	className="full-width-image"
/>

- **Optimized dark color levels:** Improvements in darker shades for clearer gradations.
- **Dark Mode optimized:** Improved consistency and harmony compared to Light Mode.

<img
	src="/assets/migration-guides/v2/colors-steps-backgrounds.png"
	alt="Color Steps and Backgrounds"
	className="full-width-image"
/>

- **Interactive Tokens:** Reworked for clearer differences between **default**, **hovered** and **pressed** states.

<img
	src="/assets/migration-guides/v2/colors-interactive-colors.png"
	alt="Interactive Colors"
	className="full-width-image"
/>

- **Removal of On-Origin colors:** The interactive colors ❌ **on-origin-hovered** and ❌ **on-origin-pressed** are removed as they do not ensure sufficient accessibility across the system.

	]}

</DBSection>

## Components: Bugfixes (Figma)

The following fixes require special attention. An overview of all other resolved bugs can be found in the **Issue Teaser** at the bottom of the page.

### Badge

- **Emphasis Strong optimization:** Using **vibrant** instead of **contrast-high** as background color for a more vivid badge appearance.

<img
	src="/assets/migration-guides/v2/migration-badge-colors.png"
	alt="Badge Colors"
	className="full-width-image"
/>

### Tag

- **Removal of Emphasis Origin:** This variant is being removed.
- **Emphasis Strong optimization:** Using **vibrant** instead of **contrast-high** as background color for a more vivid tag appearance.
- **Transition phase for design:**
  - Removed variants are initially marked as **deprecated** ❌ to allow gradual replacement.
- **Deletion with v2.1.0:** The complete removal of the Origin emphasis will happen with the next minor release.

<img
	src="/assets/migration-guides/v2/migration-tag-colors.png"
	alt="Tag Colors"
	className="full-width-image"
/>

### Form Fields (Input, Select, Textarea)

- **Unified min-width:** Consistent minimum widths for all **data input components**.
- Input specifications:
  - No icon: **db-sizing-lg**
  - 1 icon: **db-sizing-xl**
  - 2 icons: **db-sizing-2xl**
- Select specifications:
  - No icon: **db-sizing-xl**
  - 1 icon: **db-sizing-2xl**
- Textarea: **db-sizing-lg**

<img
	src="/assets/migration-guides/v2/migration-data-input.png"
	alt="Data Input Min Widths"
	className="full-width-image"
/>
