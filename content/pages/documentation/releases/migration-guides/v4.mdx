## Components

### Switch

- **Removal of the "Emphasis" property (Breaking Change)**
  The "Emphasis" property is being removed as it can cause visual conflicts with the "Validation" property. **The default variant "Weak" remains, the Strong variant is marked as deprecated and will be removed from the system.**
  In Figma, affected deprecated components are recognizable by a pink background and marked with **"❌ Switch → Strong [...] (Deprecated)"** in the name. **These components will be fully removed with the update on November 24, 2025.**

<img
	src="/assets/migration-guides/v4/switch-emphasis-fix.png"
	alt="Switch Emphasis Fix"
	style={{ maxWidth: '100%' }}
/>

<YourAction
	actions={[

- The **"Validation"** property is now available for all states. You can directly map and use the
statuses "No Validation" (default), "Invalid" and "Valid" as variants.

<img
	src="/assets/migration-guides/v4/switch-validation.png"
	alt="Switch Validation"
	style={{ maxWidth: '100%' }}
/>

- The new **"Label Position"** property supplements the existing Trailing Label (default) with the "Leading Label" variant.

<img
	src="/assets/migration-guides/v4/switch-label-leading.png"
	alt="Switch Label Position"
	style={{ maxWidth: '100%' }}
/>

- With the new **"Show Message"** property, it is now possible to display a helper message even for No Validation variants. This allows you to show helpful hints independent of the validation status.

### Notification (Figma/breaking change)

To ensure visual consistency between Figma and development, the "Show Headline" property is now represented as a **dedicated variant**. As a result, when updating the component, the headline will also appear in variants that previously had the headline hidden. It must be hidden again after the update.
With this update, the behavior in design and development becomes identical.

<img
	src="/assets/migration-guides/v4/notification-headline.png"

	style={{ maxWidth: '100%' }}
/>

<img
