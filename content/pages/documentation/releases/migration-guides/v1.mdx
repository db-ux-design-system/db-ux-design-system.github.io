## Components: Property Alignment

### General

The property alignment between design and code, started with v0.6.0, has been continued for additional components.

- **Affected components:**
  - Accordion
  - Card
  - Divider
  - Navigation
  - Popover
  - Section
  - Switch
  - Tag
  - Tooltip

- **Consistency and new properties:**
  All sub-components within a parent component have been unified.

- **New development properties (💻)**
  Properties introduced for development are marked with 💻 (e.g. Prefix, Component, No Text, Children, Placement).

- **Design-specific properties (🎨)**
  Properties relevant only to design are marked with 🎨.

- **New variants and booleans:**
  - **Disabled property (true/false):** Introduced as a dedicated variant, Interaction State = None
  - **Readonly property (true/false):** Introduced as a dedicated variant, Interaction State = None
  - **true/false** for various properties that indicate a state

- **Improved accessibility:**
  - **Text property for screen readers:** Components without visible text (e.g. Icon Button, Icon Tag, Badge without text) now include a hidden text property for screen readers.

- **Minor property name adjustments**, e.g.:
  - "Width: Auto Width/Full Width" → "Width: Auto/Full" (e.g. Button)
  - "🎨 xxx" → "Show xxx" (various components)

</DBSection>

<DBSection spacing="small" width="auto">

### Components

- **Popover**
  - In addition to Auto Width, there is now a variant for Fixed Width behavior with a maximum width
  - An additional spacing has been added (lg)

- **Section**
  - Existing widths have been renamed to t-shirt sizes and supplemented with a Small variant
  - Size has been renamed to Spacing and supplemented with a None variant

- **Tag**
  - The additional **Emphasis "Origin" has been added.**
    It enables accessible use of the tag with a more vibrant color level from the color palette.

    <img
    	src="/assets/migration-guides/v1/migration-tag-origin.png"
    	alt="Tag Emphasis Origin"
    	style={{ maxWidth: '100%' }}
    />

  - The **colors and color logic** across the different tag variants have been **adjusted and unified.** Thank you for your feedback on this! (see Rework Foundation → Color Variables)

    <img
    	src="/assets/migration-guides/v1/migration-tag-colors.png"
    	alt="Tag Colors"
    	style={{ maxWidth: '100%' }}
    />

  - A **slot has been added** to the content area of the tag to allow more flexible integration of custom content. **Please refer to the documentation for correct usage!**

    <img
    	src="/assets/migration-guides/v1/migration-tag-slot.png"
    	alt="Tag Slot Content"
    	style={{ maxWidth: '100%' }}
    />

  - The **Interactive variant** of the tag has been **structurally and content-wise reworked:** - **Split of the Interactive Tag into two variants** based on behavior: - **Interactive Toggle** (formerly Interactive): this variant represents the tag's behavior as Checkbox or Radio, displaying the states "Not selected" (checked = false) and "Selected" (checked = true) - **Interactive** (new): this variant represents the tag's behavior as Button or Link

            - For **Interactive Toggle**, the two states "Not selected" (checked = false) and "Selected" (checked = true) have been combined for better usability of the component.

            - An (optional) **Checked Status Icon has been added**. It enables better differentiation between the Interactive Toggle Tag and the Static Tag when used in parallel within an application.

    <img
    	src="/assets/migration-guides/v1/migration-tag-interactive.png"
    	alt="Tag Interactive Variants"

## Rework Foundations

### Variable Modes (Figma)

- Removal of deprecated "Density" modes such as ❌ **Tablet-XXX** and ❌ **Mobile-XXX.**
- Renaming of "Display Mode" to "Mode"

### Color Variables

- The **selection of brand colors as origin colors** has been reworked and harmonized by choosing not just level 500, but various appropriate gradations. All origin colors have been adjusted to meet **accessibility requirements and component usage**, ensuring a **consistent color appearance of origin and on-origin colors**, particularly through the combination of dark colors on light backgrounds. This achieves a **consistent representation** for all semantic and additional colors and **improves differentiation from brand red.**
