## Components: Bugfixes (Figma)

The following fixes require special attention. An overview of all other resolved bugs can be found in the **Issue Teaser** at the bottom of the page.

### Accordion

- **The Accordion Item List behavior has been corrected:** Previously, the item list in the Accordion was implemented using a central helper component with slots. This caused errors due to a Figma bug when using the Accordion component. With this update, the item list is now implemented directly within the Accordion component with the Accordion Items.

<img
	src="/assets/migration-guides/v3/accordion-bug.png"
	alt="Accordion Bug"
	style={{ maxWidth: '100%' }}
/>

<img
	src="/assets/migration-guides/v3/accordion-list.png"
	alt="Accordion Item List as Subcomponent"
	style={{ maxWidth: '100%' }}
/>

- Optimization of prototyping behavior.

<YourAction
	actions={[
		'Review your deployed Navigation components.',
		'Layout corrections: Adjust layouts if Accordion List changes cause issues with deployed Accordions.',
	]}
/>

**❓ How do I identify the reworked component?**

Switch to the second level of the component. If the right sidebar shows ".Slot 🟪 Vertical" or ".Slot 🟪 Horizontal", you are using the outdated component.

<img
	src="/assets/migration-guides/v3/accordion-slot.png"
	alt="Outdated Accordion with Slot List"
	style={{ maxWidth: '100%' }}
/>

**❓ How do I update an already deployed Accordion to the latest variant?**

Switch to the first level of the component and reset it.

⚠️ **Note:** Due to the **Breaking Changes**, you will unfortunately need to redo your settings and re-insert all content.

<img
	src="/assets/migration-guides/v3/accordion-reset.png"
	alt="Reset Accordion Component"
	style={{ maxWidth: '100%' }}
/>

</DBSection>
