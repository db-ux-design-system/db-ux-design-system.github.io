Active describes an element that is currently checked, selected or highlighted in the interface. This state is typically created by giving the element a higher visual hierarchy than its default enabled state. This pattern does not prescribe a single fixed style, but describes a set of methods that can be used individually or in combination.

Depending on context, hierarchy, and interface density, the active state can be expressed via the container (surface, color, border) or via the content (typography, icon). The following variants illustrate common approaches that can be combined consistently and adapted across different components.

## Container

### Surface

The first option is to increase the visual emphasis of the container. This can be achieved through stronger contrast or even inverted foreground and background, making the active state stand out immediately.

### Color

Another option is an adjustment of the container color. A shifted or tinted background distinguishes the active state from inactive elements without dominating the overall layout. This can be easily applied by changing the data-color.

### Border

The active state can also be shown through a more prominent border, for example with increased stroke width or change of color. The changed outline makes the active element clearly recognizable, even when surfaces remain visually restrained.

## Content

### Typography

Instead of changing the container, the active state can also be highlighted through typography. A higher font weight or other typographic accents ensure that the label of the active element appears clearly more dominant in direct comparison.

### Content

Finally, the active state can be expressed by changing or extending the content. An additional icon, such as a checkmark, or a modified label makes the selection semantically clear and reinforces the visual emphasis.
