## How does Code Connect work?

Code Connect is a Figma feature that links design components directly to their code implementations. Developers see the exact code snippet for a selected component configuration right inside Figma's Dev Mode.

In the DB UX Design System, component properties in Figma drive the Code Connect output. There are two categories:

1. **Design properties:** affect the visual appearance and are always visible on the first level (e.g. Size, Label, Disabled).
2. **⚙️ Code Connect properties:** only relevant for code generation, not for the visual design. If configurable, they appear on the first level marked with a ⚙️ icon. If not configurable, they are handled in the background.

</DBSection>

## For Designers

As a designer, you work with both types of properties when configuring a component instance.

### Design-relevant properties

These are the properties you see directly when selecting a component instance in Figma. They affect the visual appearance of the component.

You configure these as usual. They influence both the design and the generated code output.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>⚙️ Code Connect properties</h3>

Code Connect properties are marked with a **⚙️** icon and are only relevant for code generation. They do not change the visual design.

If a component has configurable Code Connect properties, they appear directly on the first level at the end of the properties list. You can recognize them by the ⚙️ prefix.

However, not all components have configurable Code Connect properties. Some components (like the Outline Button) contain a **⚙️ Code Connect** layer internally that passes "blind" properties to the code output (e.g. `variant="outlined"`). Since there is nothing to adjust, no ⚙️ properties are shown on the first level.

## For Developers

As a developer, Code Connect gives you direct access to the correct code snippet for any component configuration, right inside Figma's Dev Mode.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>How it works</h3>

Code Connect maps Figma component properties to framework-specific code snippets. When a designer configures a component (both design-relevant and Code Connect properties), the Dev Mode panel shows the corresponding code that matches that exact configuration.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>Using Code Connect in Dev Mode</h3>

1. Open a Figma file and switch to **Dev Mode** (toggle in the top toolbar).
2. Select a component instance on the canvas.
3. In the **Inspect panel** on the right, look for the **Code Connect** section (marked as "Connected").
4. Select your target framework (e.g. React) from the dropdown.
5. The generated code snippet reflects the current property configuration, including values set via ⚙️ Code Connect properties.

<div style={{ marginTop: 'var(--db-spacing-fixed-sm)' }}>

| Property type            | Configured by                    | Example properties                     | Reflected in code  |
| ------------------------ | -------------------------------- | -------------------------------------- | ------------------ |
| Design-relevant          | Designer (component first level) | `size`, `label`, `disabled`            | ✅ Yes             |
| Code-only (configurable) | Designer (⚙️ on first level)     | `required`, `checked`, `indeterminate` | ✅ Yes             |
| Code-only (automatic)    | Determined by variant            | `variant`, `noText`                    | ✅ Yes (automatic) |

Both types of properties are reflected in the generated code output. The distinction only matters for the design workflow. In code, all properties are treated equally.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>Default values</h3>

⚙️ Code Connect properties have sensible defaults pre-configured. If a designer hasn't explicitly changed a value, the default is used in the code output. This means you can trust the generated snippet to be complete and accurate for the selected configuration.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>Code Connect configuration</h3>

Code Connect is configured via `.figma.ts` files in the component source code. These files define the mapping between Figma properties and code props using a custom abstraction layer.

```ts
// checkbox.figma.ts
import { FigmaCodeConnect, FigmaProp } from '../../../shared/figma';

export type FigmaCheckboxProps = {
	label?: string;
	size?: string;
	required?: boolean;
	showRequiredAsterisk?: boolean;
	disabled?: boolean;
	checked?: boolean;
	indeterminate?: boolean;
	validation?: string;
	message?: string;
};

const checkboxProps: Record<string, FigmaProp> = {
	label: { type: 'textContent', key: 'Label' },
	size: {
		type: 'enum',
		key: 'Size',
		value: { Medium: 'medium', Small: 'small' },
	},
	required: { type: 'boolean', key: 'Required' },
	showRequiredAsterisk: { type: 'boolean', key: 'Show Required Asterisk' },
	disabled: { type: 'boolean', key: 'Disabled' },
	checked: { type: 'boolean', key: 'Checked' },
	indeterminate: { type: 'boolean', key: 'Indeterminate' },
	validation: {
		type: 'enum',
		key: 'Validation',
		value: {
			'(Def) No Validation': 'no-validation',
			Valid: 'valid',
			Invalid: 'invalid',
		},
	},
	message: {
		type: 'validationMessage',
		key: 'Text',
		conditionProp: 'validation',
		map: {
			invalid: 'invalidMessage',
			valid: 'validMessage',
			default: 'message',
		},
	},
};

export const checkboxes: FigmaCodeConnect = {
	urls: [
		'https://www.figma.com/design/FIGMA_FILE?node-id=2068:3423',
		// ... more variant URLs
	],
	props: checkboxProps,
};
```

Each `.figma.ts` file exports a `FigmaCodeConnect` object containing:

- **urls**: Links to the Figma component variants this configuration applies to.
- **props**: A mapping of code prop names to their Figma property definitions (type, key, value mappings).
