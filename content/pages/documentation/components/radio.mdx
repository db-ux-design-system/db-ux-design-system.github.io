## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

###### Radio vs Checkbox

Use radio buttons when users must choose exactly one option from a group. If multiple selections are allowed, use Checkbox instead.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/kO9Hk3PMKNvggHWUNQZNFf/Doku--RadioRadio-%E2%9C%85?node-id=1-1132&embed-host=share"
		description="Use radio buttons for mutually exclusive options."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/kO9Hk3PMKNvggHWUNQZNFf/Doku--RadioRadio-%E2%9C%85?node-id=1-1133&embed-host=share"
		description="Don’t use radio buttons for multi-select scenarios."
	/>
</div>

###### Radio vs Select

Use Radio when the number of options is small and users benefit from seeing all options at once. Use Select when options are many or screen space is limited.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/kO9Hk3PMKNvggHWUNQZNFf/Doku--RadioRadio-%E2%9C%85?node-id=3-1110&embed-host=share"
		description="Use radio buttons for 2–5 options that users should compare quickly."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/kO9Hk3PMKNvggHWUNQZNFf/Doku--RadioRadio-%E2%9C%85?node-id=3-1112&embed-host=share"
		description="Don't use radio buttons for long lists where scrolling and scanning becomes hard."
	/>
</div>

###### Labels

Labels must be concise and clearly distinct from each other. Keep wording parallel and avoid vague or overlapping terms.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/kO9Hk3PMKNvggHWUNQZNFf/Doku--RadioRadio-%E2%9C%85?node-id=3-1113&embed-host=share"
		description="Use clear and concise labels."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/kO9Hk3PMKNvggHWUNQZNFf/Doku--RadioRadio-%E2%9C%85?node-id=3-1114&embed-host=share"
		description="Avoid unclear labels or double negatives that can confuse."
	/>
</div>

###### Grouping

Prefer vertical stacking for readability and accessibility. Use horizontal layout only when there are few, short options and sufficient space.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"

	/>
	<GuidelineExample

		description="Don’t place many options in a single horizontal row where labels wrap or alignment breaks."
	/>
</div>
