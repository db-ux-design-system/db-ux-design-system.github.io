## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

###### Use Radio for single-choice sets

Use radio buttons when users must choose exactly one option from a group. If multiple selections are allowed, use Checkbox instead.

	style={{
		display: 'flex',

		description='Use radio buttons for multi-select scenarios like "Email + SMS + Post".'
	/>
</div>

###### Prefer Radio over Select when comparison matters

Use Radio when the number of options is small and users benefit from seeing all options at once. Use Select when options are many or screen space is limited.

	style={{

###### Write short, unambiguous labels

Labels must be concise and clearly distinct from each other. Keep wording parallel and avoid vague or overlapping terms.

###### Arrange options for easy scanning

Prefer vertical stacking for readability and accessibility. Use horizontal layout only when there are few, short options and sufficient space.

	style={{
		display: 'flex',

		description="Place many options in a single horizontal row where labels wrap or alignment breaks."
	/>
</div>

###### Set and communicate a sensible default

If a safe default exists, preselect it to reduce effort. If no default is appropriate, start with no selection and validate on submit as needed.

	style={{

	/>

		description="Preselect an option that implies consent or a high-impact choice without user intent."
	/>
</div>

###### Accessibility and screenreader text

Every radio must have an accessible name, and the group should have a clear group label. If the visible label is hidden, provide screenreader label text. Ensure keyboard navigation works as expected within the group.
