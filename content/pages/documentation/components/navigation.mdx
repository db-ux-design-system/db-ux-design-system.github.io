## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

###### Destinations

Use Navigation items to take users to pages or views. For operations (create, save, export), use Buttons or Menus, not navigation items.

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',
		marginBottom: 'var(--db-spacing-fixed-2xl)',
	}}
>
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/lDMxw7YqQoJaWQdBlG7kwB/Doku--Navigation?node-id=1-1132&embed-host=share"
		description="Use navigation items for stable destinations like 'Overview', 'Reports', 'Settings', displayed as a consistent list/tab set with a clear active item."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/lDMxw7YqQoJaWQdBlG7kwB/Doku--Navigation?node-id=1-1133&embed-host=share"
		description="Don't use navigation items for one-off actions like 'Download now' or 'Submit', shown as destinations instead of an action control."
	/>
</div>

###### Variant

Choose horizontal navigation for shallow structures with enough space. Use vertical navigation for sidebars, longer lists, or deeper hierarchies that must stay visible.

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',
		marginBottom: 'var(--db-spacing-fixed-2xl)',
	}}
>
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/lDMxw7YqQoJaWQdBlG7kwB/Doku--Navigation?node-id=3-1110&embed-host=share"
		description="Use horizontal navigation for top-level destinations and vertical navigation for section navigation, with labels fully visible and easy to scan."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/lDMxw7YqQoJaWQdBlG7kwB/Doku--Navigation?node-id=3-1112&embed-host=share"
		description="Don't force horizontal navigation into tight spaces where labels truncate, wrap unpredictably, or reduce scannability."
	/>
</div>

###### Labels

Keep labels short, clear, and parallel in wording. Avoid repeating the same prefix across items and keep the set stable over time.

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',
		marginBottom: 'var(--db-spacing-fixed-2xl)',
	}}
>
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/lDMxw7YqQoJaWQdBlG7kwB/Doku--Navigation?node-id=3-1113&embed-host=share"
		description="Use concise, descriptive labels that fit common viewports without truncation, aligned to a consistent text style and casing."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/lDMxw7YqQoJaWQdBlG7kwB/Doku--Navigation?node-id=3-1114&embed-host=share"
		description="Avoid using long phrases, mixed grammar/casing, or frequently changing item names/order in ways that break recognition."
	/>
</div>

###### Hierarchy

Keep hierarchy shallow and predictable. If there are too many items, use an overflow pattern (e.g., 'More') or a menu-based expansion.

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',
		marginBottom: 'var(--db-spacing-fixed-2xl)',
