## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

###### Switch vs Checkbox

Use Switch only when toggling should take effect right away. If users are selecting options to be applied later (e.g., on submit), use Checkbox instead.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/LIYooyKYqaul7irlJKDflt/Doku--Switch-%E2%9C%85?node-id=1-1132&embed-host=share&scaling=scale-down-width"
		description="Use a switch if the change applies instantly."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/LIYooyKYqaul7irlJKDflt/Doku--Switch-%E2%9C%85?node-id=1-1133&embed-host=share&scaling=scale-down-width"
		description="Don't use a switch inside a form where changes only apply after submitting."
	/>
</div>

###### Labels

Keep labels concise and avoid negative wording that can create confusion. If helpful, you can also use the label dynamically and set it individually for checked and unchecked states.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/LIYooyKYqaul7irlJKDflt/Doku--Switch-%E2%9C%85?node-id=3-1110&embed-host=share&scaling=scale-down-width"
		description="Use concise labels instead of negative phrasing."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/LIYooyKYqaul7irlJKDflt/Doku--Switch-%E2%9C%85?node-id=3-1112&embed-host=share&scaling=scale-down-width"
		description="Don't use wording that becomes ambiguous when toggled."
	/>
</div>

###### Visual Aid

Icons can be used as visual aids to make the status of a switch easier to understand. Use only clear, unambiguous icons that match the label and context.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/LIYooyKYqaul7irlJKDflt/Doku--Switch-%E2%9C%85?node-id=3-1113&embed-host=share&scaling=scale-down-width"
		description="Use a simple icon that directly corresponds to the status."
