## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

### Use Switch for immediate state changes

Use Switch only when toggling should take effect right away. If users are selecting options to be applied later (e.g., on submit), use Checkbox instead.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/LIYooyKYqaul7irlJKDflt/Doku--Switch-%E2%9C%85?node-id=1-1132&embed-host=share&scaling=scale-down-width"
		description='Use a switch for "Enable automatic updates" when the change applies instantly.'
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/LIYooyKYqaul7irlJKDflt/Doku--Switch-%E2%9C%85?node-id=1-1133&embed-host=share&scaling=scale-down-width"
		description='Use a switch inside a form where changes only apply after pressing "Save".'
	/>
</div>

### Make state change clear and observable

Users must be able to immediately understand whether the switch is on or off and see the effect of the change (UI update, enabled feature, or confirmation feedback).

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/LIYooyKYqaul7irlJKDflt/Doku--Switch-%E2%9C%85?node-id=3-1110&embed-host=share&scaling=scale-down-width"
		description="Reflect the new state visually and, if needed, provide lightweight feedback (e.g., infotext/notification)."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/LIYooyKYqaul7irlJKDflt/Doku--Switch-%E2%9C%85?node-id=3-1112&embed-host=share&scaling=scale-down-width"
		description="Toggle a switch without visible state clarity or without any observable outcome."
	/>
</div>

### Write short, positive label text

Keep labels concise and avoid negative wording that can create confusion ("Don't", double negatives). If helpful, use dynamic labels that describe the current state in plain language.
