## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

###### Textarea vs Input

Use Textarea when users are expected to enter multiple lines or a longer message. For short, single-line values, use Input.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/2gF67xeMuiyGIlW6lHHNC1/Doku--Textarea?node-id=1-1132&embed-host=share"
		description="Use the Textarea for comments, messages, descriptions, and feedback."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/2gF67xeMuiyGIlW6lHHNC1/Doku--Textarea?node-id=1-1133&embed-host=share"
		description="Don't use the Textarea for short values like name, email, or city (use Input instead)."
	/>
</div>

###### Labels

Provide a clear label that describes what to write. Do not replace the label with placeholder text; placeholders disappear when typing.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/2gF67xeMuiyGIlW6lHHNC1/Doku--Textarea?node-id=3-1110&embed-host=share"
		description="Use a short, specific label placed consistently above or floating. Also consider surrounding form components."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/2gF67xeMuiyGIlW6lHHNC1/Doku--Textarea?node-id=3-1112&embed-host=share"
		description="Don't use multi-line labels or punctuate them."
	/>
</div>

###### Height

Choose a height that matches the expected amount of text input. Avoid very small textareas that force scrolling immediately and very tall ones that dominate the page. Use a "Fixed" field size (default: four rows, customizable) or the "Content" variant, which starts with one row and expands with input.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/2gF67xeMuiyGIlW6lHHNC1/Doku--Textarea?node-id=3-1113&embed-host=share"
		description="Start with an appropriate height that matches the expected input and the context of use."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/2gF67xeMuiyGIlW6lHHNC1/Doku--Textarea?node-id=3-1114&embed-host=share"
		description="Avoid a small Textarea if a lot of input is expected and there is sufficient space."
	/>
</div>

###### Message

Helper text prevents errors (format, tone, limits). Validation messages explain what's wrong and how to fix it. Keep both concise and actionable; do not duplicate the label.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/2gF67xeMuiyGIlW6lHHNC1/Doku--Textarea?node-id=3-1116&embed-host=share"
		description="Provide a brief helper hint and a clear error message when invalid."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/2gF67xeMuiyGIlW6lHHNC1/Doku--Textarea?node-id=3-1117&embed-host=share"
		description="Don't use long paragraphs, vague errors ('Invalid'), or put essential requirements only in placeholder text."
	/>
</div>
