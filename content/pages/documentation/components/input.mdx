## Guidelines

### Input Type

Use Input for free text entry. If users must choose from predefined options, use a selection component instead (e.g., Select, Autocomplete, Radio/Checkbox depending on the case).

		description="Use Input when the user can type any value and the expected entry is short."
	/>

### Label text

Every Input must have a clear, descriptive label that explains what to enter. Keep labels short and specific; avoid vague labels. Don't replace the label with a placeholder.

<div className="guideline-examples">
	<GuidelineExample

		description="Place a visible label directly associated with the field (above or consistently aligned)."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/zJPzoYQJB2bFhAnceth2dm/Doku--Input?node-id=3-1112&embed-host=share&scaling=scale-down-width"
		description="Don’t use long descriptive labels."
	/>
</div>

### Placeholders

Placeholder text is optional and should provide an example format/value, not instructions or required context. Placeholders must not contain critical information because they disappear when users type.

		description='Use placeholder for an example (e.g., "name@company.com") while the label carries meaning.'

### Message

Use helper text to prevent errors (format, constraints, expectations). Use validation messages to explain what is wrong and how to fix it. Keep messages concise and actionable.

	<GuidelineExample

		description="Show a short helper message for format guidance and an explicit error message when invalid."
	/>

		figmaUrl="https://embed.figma.com/design/zJPzoYQJB2bFhAnceth2dm/Doku--Input?node-id=3-1117&embed-host=share&scaling=scale-down-width"

			'Don\'t show long paragraphs, duplicate the label as helper/error text, or use unclear errors (e.g., "Invalid input").'

### Field Width

Inputs must align to the layout grid and maintain consistent spacing between label, field, helper/validation text, and adjacent fields. Avoid stretching labels or misaligning fields in responsive layouts.

		description="Keep label, input, and helper/validation text aligned and consistently spaced across breakpoints."

		figmaUrl="https://embed.figma.com/design/zJPzoYQJB2bFhAnceth2dm/Doku--Input?node-id=3-1118&embed-host=share&scaling=scale-down-width"

	/>

### Accessibility

Ensure every Input has an accessible name that matches the visible label. If a label is visually hidden, provide an equivalent screenreader label. Required/optional indicators must be announced meaningfully.
