Input captures short, single-line user text entries in forms and dialogs. It should be predictable, clearly labeled, and provide guidance and validation feedback when needed.

- Collect short text values (e.g., name, city, email, IBAN depending on product rules)
- Support structured entry with formatting hints (e.g., "No spaces", "Use format DD.MM.YYYY")
- Provide immediate or on-submit feedback for validity (success/error) with helper text

## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

###### Right data input type

Use Input for short, single-line free text entry. Use Textarea if users are expected to enter multiple lines or longer messages. If users must choose from predefined options, use a selection component instead (e.g., Select, Radio/Checkbox depending on the case).

###### Label text

Provide a clear and short label that describes what to write. Do not replace the label with placeholder text; placeholders disappear when typing.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"

		description="Place a visible label directly associated with the field and keep it short an precise."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/zJPzoYQJB2bFhAnceth2dm/Doku--Input?node-id=3-1112&embed-host=share&scaling=scale-down-width"
		description="Don't use long descriptive labels."
	/>
</div>

###### Show/hide label

Use visible labels (above or floating) as the default. Only use hidden labels in exceptional cases where space is extremely constrained. When using hidden labels, always provide screenreader text so assistive technology users know what to enter.

	/>

</div>

###### Placeholders as examples

Placeholder text is optional and should provide an example format/value, not instructions or required context. Placeholders must not contain critical information because they disappear when users type.

###### Message

Use helper text to prevent errors (format, constraints, expectations). Use validation messages to explain what is wrong and how to fix it. Keep messages concise and actionable.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/zJPzoYQJB2bFhAnceth2dm/Doku--Input?node-id=3-1116&embed-host=share&scaling=scale-down-width"
		description="Show a short helper message for format guidance and an explicit error message when invalid."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/zJPzoYQJB2bFhAnceth2dm/Doku--Input?node-id=3-1117&embed-host=share&scaling=scale-down-width"

	/>
</div>

###### Field width

Input width should reflect the expected content length and align to the layout grid. Avoid stretching inputs to fill the entire screen width. Size them appropriately for realistic content.
