## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

###### Hierarchy and primary CTA

Establish a clear hierarchy so users can see which actions matter most. Use the Brand Button as the strongest CTA to drive conversion and use it only once per viewport.

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',
		marginBottom: 'var(--db-spacing-fixed-2xl)',
	}}
>
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/7ZUEgSRPJtyT8FKRT97Dzq/Doku--Button?node-id=3-852&embed-host=share"
		description="Use one primary button to highlight the main action. Place secondary actions next to it with lower emphasis."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/7ZUEgSRPJtyT8FKRT97Dzq/Doku--Button?node-id=3-863&embed-host=share"
		description="Do not place two primary buttons next to each other or in the same viewport. Competing primary actions make it harder to decide what to do."
	/>
</div>

###### Labels

Keep labels short, specific, and action‑oriented. Auto‑width buttons are single‑line by default; do not insert manual line breaks. Full‑width buttons may wrap only in edge cases (e.g., localization or narrow screens).

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',
		marginBottom: 'var(--db-spacing-fixed-2xl)',
	}}
>
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/7ZUEgSRPJtyT8FKRT97Dzq/Doku--Button?node-id=3-1125&embed-host=share"
		description="Use concise, single‑line labels."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/7ZUEgSRPJtyT8FKRT97Dzq/Doku--Button?node-id=3-1137&embed-host=share"
		description="Manually break lines or use long, complex label text."
	/>
</div>

###### Icons

Use intuitive, universally understood icons. Add at most one icon inside a button and choose a suitable position (leading or trailing). Icon‑only buttons suit toolbars, navigation, and mobile; always show a tooltip on hover.

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',
		marginBottom: 'var(--db-spacing-fixed-2xl)',
	}}
>
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/7ZUEgSRPJtyT8FKRT97Dzq/Doku--Button?node-id=3-1152&embed-host=share"
		description="Use clear icons with a hover tooltip for icon‑only buttons"
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/7ZUEgSRPJtyT8FKRT97Dzq/Doku--Button?node-id=3-1460&embed-host=share"
		description="Don't place two icons inside one button"
	/>
</div>

###### Layout and responsive

Place related buttons horizontally if space allows. Use auto width by default; switch to full width in responsive/mobile layouts. The button must never be narrower than its content.

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',
		marginBottom: 'var(--db-spacing-fixed-2xl)',
	}}
>
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/7ZUEgSRPJtyT8FKRT97Dzq/Doku--Button?node-id=3-1594&embed-host=share"
		description="Preferably align buttons side by side if space allows"
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/7ZUEgSRPJtyT8FKRT97Dzq/Doku--Button?node-id=3-3298&embed-host=share"
		description="Don't Stack buttons vertically when horizontal space exists, or let content overflow the button."
	/>
</div>

###### Button vs Link

Use buttons to trigger actions that change state. Use links to navigate to other pages or views.

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',
		marginBottom: 'var(--db-spacing-fixed-2xl)',
	}}
>
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/7ZUEgSRPJtyT8FKRT97Dzq/Doku--Button?node-id=3-3649&embed-host=share"
		description="Use a button to perform an in‑app action that changes state and a link to navigate to another page or view"
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/7ZUEgSRPJtyT8FKRT97Dzq/Doku--Button?node-id=3-3823&embed-host=share"
		description="Don't use a button for page navigation"
	/>
</div>

###### Accessibility

The small button in functional density has a height of 20 px. To meet accessibility requirements, its interactive target must be at least 24×24 px in size. You can achieve this by ensuring there is sufficient space around the button. No other interactive elements may occupy or overlap this target area.

	style={{
		display: 'flex',

	/>
