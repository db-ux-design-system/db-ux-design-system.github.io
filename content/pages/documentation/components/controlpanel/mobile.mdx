- Position Top: Default horizontal bar with Brand and burger menu triggering a Right Drawer
- Position Bottom: Thumb-reachable bar with Brand, burger menu, and optional Primary Actions triggering a Bottom Drawer
- Position FlatIcon: Tab bar with 2–5 icon-based navigation destinations for direct access without a menu
- Drawer navigation uses the same Popover pattern with level-based navigation and back buttons

## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

### Position

Select the ControlPanel position based on the app type and primary interaction pattern. Top is the default for content-focused apps. Bottom works for app-like experiences where navigation actions need to be within thumb reach.

<div className="guideline-examples">
	<GuidelineExample

		description="Use Top for content-focused applications where navigation is secondary to reading or consuming content."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/pElrqVUyojrzYzSagyJPS6/Doku--ControlPanel?node-id=TODO-dont&embed-host=share"
		description="Do not use Bottom when the application has no primary actions to place in the thumb zone. The bottom bar wastes space without added benefit."
	/>
</div>

### FlatIcon Usage

FlatIcon (tab bar) works best for apps with a small set of equally important top-level destinations. It provides instant access without opening a Drawer but has limited capacity.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/pElrqVUyojrzYzSagyJPS6/Doku--ControlPanel?node-id=TODO-do&embed-host=share"
		description="Use FlatIcon when the application has 2–5 equally weighted top-level sections that users switch between frequently."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/pElrqVUyojrzYzSagyJPS6/Doku--ControlPanel?node-id=TODO-dont&embed-host=share"
		description="Do not use FlatIcon when navigation items require explanatory labels or when there are more than 5 destinations. Use Top or Bottom with a Drawer instead."
	/>
</div>

### Density

The mobile ControlPanel only supports Regular density. Touch targets need sufficient size for reliable finger interaction on mobile devices.
