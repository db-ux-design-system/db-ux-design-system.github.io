Notification communicates a brief message about an event, state change, or system feedback without blocking the user. It should be timely, concise, and clearly indicate intent (info, success, warning, error) with predictable dismissal behavior.

- Confirm an action or state change (e.g., "Saved", "Filter applied")
- Inform about a non-blocking system event (e.g., "Sync in progress", "New version available")
- Warn or report an issue that needs attention without requiring an immediate modal decision

## Usage Hints

<CollapsibleWrapper collapsedHeight="320px">

###### Usage

Use notification for feedback that lets users continue their task. If the message requires an immediate decision, confirmation, or blocks a critical flow, use a Dialog instead.

<div
	style={{
		display: 'flex',

		description="Show a notification after a successful save or background task result, appearing in the standard corner stack with a calm success/info style."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/CI4pJdjNIxXeNS5RFUqjpK/Doku--Notification?node-id=1-1133&embed-host=share"
		description="Don't use a notification to confirm irreversible actions or force a decision, especially when the user must respond to proceed."
	/>
</div>

###### Semantics

Use semantic variants consistently: Successful for completion, informational for new updates, warning for risk/attention, critical for failures that need correction. Ensure the message is understandable without color.

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',
		marginBottom: 'var(--db-spacing-fixed-2xl)',
	}}
>

		variant="do"
		figmaUrl="https://embed.figma.com/design/CI4pJdjNIxXeNS5RFUqjpK/Doku--Notification?node-id=3-1110&embed-host=share"
		description="Use an critical notification for failed actions, stating what happened and the next step, with an error icon and clear label text."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/CI4pJdjNIxXeNS5RFUqjpK/Doku--Notification?node-id=3-1112&embed-host=share"
		description="Don't use successful styling for neutral updates or rely on color alone to communicate urgency or meaning."
	/>
</div>

###### Content

Keep messages short and actionable. Include what happened and, if relevant, the next step. If details are needed, link to a dedicated surface (page, panel, popover).

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',

		variant="do"
		figmaUrl="https://embed.figma.com/design/CI4pJdjNIxXeNS5RFUqjpK/Doku--Notification?node-id=3-1113&embed-host=share"
		description="Use an optional short title + one concise sentence, and one clear action link/button when needed, laid out in a single clean row."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/CI4pJdjNIxXeNS5RFUqjpK/Doku--Notification?node-id=3-1114&embed-host=share"
		description="Don't use multi-paragraph explanations or combine multiple unrelated messages into one notification block."
	/>
</div>

###### Timing

Match dismissal behavior to urgency. Auto-dismiss only for informational or success messages. Warnings and errors should remain until dismissed or resolved. Place the close control consistently when shown.

<div
	style={{
		display: 'flex',
		gap: 'var(--db-spacing-fixed-md)',

	}}
>

		variant="do"
		figmaUrl="https://embed.figma.com/design/CI4pJdjNIxXeNS5RFUqjpK/Doku--Notification?node-id=3-1116&embed-host=share"
		description="Auto-dismiss success/info after the standard duration; keep warning/error persistent with a visible close button."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/CI4pJdjNIxXeNS5RFUqjpK/Doku--Notification?node-id=3-1117&embed-host=share"
		description="Don't auto-dismiss warnings/errors before users can read or act, or vary close button placement between variants."
	/>
</div>

###### Placement

Place notifications where they are noticeable but do not cover primary controls. When multiple notifications appear, stack them consistently and limit how many are shown at once.
