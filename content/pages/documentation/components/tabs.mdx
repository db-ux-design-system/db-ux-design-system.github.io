## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

### Usage

Use Tabs when users need to switch between a small number of related sections on the same hierarchy level. For longer, linear processes where users must proceed step by step, use a stepper or wizard instead.

	<GuidelineExample

		description="Use Tabs to organize related sections that can be reached in any order."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/dRWdZQ9EjryTothOHsxeDv/Doku--Tabs?node-id=1-1133&embed-host=share"
		description="Don't use Tabs to represent mandatory steps in a process."
	/>
</div>

### Placement

Tabs should be used for top-level content sections within a page, not for global or app-wide navigation. Avoid nesting multiple tab sets in the same area to prevent confusion and complexity.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/dRWdZQ9EjryTothOHsxeDv/Doku--Tabs?node-id=3-1110&embed-host=share"
		description="Use Tabs to switch between closely related sections within one page and place them directly above the corresponding content."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/dRWdZQ9EjryTothOHsxeDv/Doku--Tabs?node-id=3-1112&embed-host=share"
		description="Don't use Tabs as a replacement for main navigation or stack multiple levels of tabs on the same screen."
	/>
</div>

### Labels

Tab labels should be short, scannable, and clearly indicate what content is shown when selected. Keep them to a single line and avoid wrapping.
