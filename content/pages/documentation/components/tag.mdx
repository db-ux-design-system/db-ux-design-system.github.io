Tag is a compact label used to categorize, highlight attributes, or represent selections. It can be static for display or interactive for filtering and selection, with optional removal where appropriate.

- Display short categories or attributes (e.g., "New", "Marketing", "Internal")
- Enable filtering/selection in lists or search results (interactive tags)
- Represent selected items that can be removed (e.g., applied filters)

## Usage Hints

###### Behavior

Tags should be used according to their behavior: Static for non-interactive categorization, Interactive (Button / Link) for context-related actions, Interactive (Checkbox / Radio) for filtering or selection, and Removable for allowing users to delete selections.

Try to keep the content of the tag as short and concise as possible. Tags should always remain single-line.

Combining the different emphases of the tag creates a clear visual hierarchy that highlights important information with the strong emphasis, while the weak emphasis provides subtle support to subtly mark secondary data.

###### Tag vs Badge

Use tags to categorize, filter, or represent selectable items. Use badges for counts or small numeric/status indicators attached to another element (e.g., notifications on an icon).

###### Accessibility

The interactive tag in "Functional" density has a height of 20 px. To meet accessibility requirements, its clickable area must be at least 24 px high, without overlapping other interactive elements. This can be achieved either by the tag's size or by providing sufficient space around it.
