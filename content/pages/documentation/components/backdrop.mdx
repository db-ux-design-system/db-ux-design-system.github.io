<DBSection id="backdrop--use-cases" spacing="none" width="large">

Backdrop is a helper component that dims content behind an overlay to direct attention to the foreground (e.g., modal, dialog, drawer). It separates active content from background content and can optionally block interaction with the background.

- Highlight an open overlay (e.g., Dialog, Modal, Drawer)
- Dim background during a blocking loading state or process
- Visually separate focused content from the rest of the page

## Implementation

There is no backdrop component. If you need a backdrop like the one in DBDrawer you should apply those styles:

<CollapsibleWrapper collapsedHeight="320px">

### CSS

```css
.my-backdrop {
	background-color: color(
		from var(--db-adaptive-on-bg-basic-emphasis-100-default) srgb r g b / var(--db-opacity-lg)
	);
}

dialog::backdrop {
	background-color: color(
		from var(--db-adaptive-on-bg-basic-emphasis-100-default) srgb r g b / var(--db-opacity-lg)
	);
}
```

### SCSS

```scss
@use '@db-ux/core-foundations/build/styles/colors';
@use '@db-ux/core-foundations/build/styles/variables';

.my-backdrop {
	background-color: color(
		from #{colors.$db-adaptive-on-bg-basic-emphasis-100-default} srgb r g b /
			#{variables.$db-opacity-lg}
	);
}

/* OR */

dialog::backdrop {
	background-color: color(
		from #{colors.$db-adaptive-on-bg-basic-emphasis-100-default} srgb r g b /
			#{variables.$db-opacity-lg}
	);
}
```

**Note:** If you want to use the weak variant you should use the `db-opacity-sm` variable.

## Guidelines

### Focus attention

Use backdrop when the user should focus on a single active UI layer and has to complete the task in the overlay to continue on the main page. If users must keep working with the background, avoid Backdrop and use a non-blocking pattern instead.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/tREBXQmJlK75D2ZBjtuBx9/Doku--Backdrop?node-id=1-1132&embed-host=share"
		description="Use backdrop behind a dialog or drawer to clearly indicate an active overlay layer."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/tREBXQmJlK75D2ZBjtuBx9/Doku--Backdrop?node-id=1-1133&embed-host=share"
		description="Don't use backdrop for decorative emphasis when no overlay is present."
	/>
</div>

### Context

Backdrop should reduce distraction, not remove necessary context. If users need to reference background information, consider an alternative (side panel, inline expansion, or non-modal popover).

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/tREBXQmJlK75D2ZBjtuBx9/Doku--Backdrop?node-id=3-1110&embed-host=share"
		description="Use backdrop only when the user can complete the task without needing background content."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/tREBXQmJlK75D2ZBjtuBx9/Doku--Backdrop?node-id=3-1112&embed-host=share"
		description="Avoid unnecessary layering unless there is a clear need to darken the background."
	/>
</div>

### Accessibility

Ensure foreground content has sufficient contrast against the dimmed background. When the backdrop indicates a modal state, keep focus within the active overlay and ensure assistive tech cannot navigate background content.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"

		description="Only enable interactivity in the foreground."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/tREBXQmJlK75D2ZBjtuBx9/Doku--Backdrop?node-id=3-1114&embed-host=share"
		description="Don't use the background without placing the content on an element with sufficient contrast."
	/>
</div>
