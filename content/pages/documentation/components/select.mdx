## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

###### Use Select for single-choice, predefined options

Use Select when the user must choose exactly one option from a list. If users need to choose multiple options, use Checkbox or a multi-select pattern. If users benefit from seeing all options, consider Radio.

		description="Use Select for one choice from a predefined set where space is limited or options are moderate in count."
	/>
	<GuidelineExample

		description="Use Select for multi-select needs or when users must compare all options at once."
	/>
</div>

###### Keep dropdown options clear and manageable

Options must be easy to scan and understand. Use concise labels, consistent capitalization, and avoid duplicates. If the list becomes long or needs search, use an Autocomplete pattern instead.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/HmUqDoWeEMtoZSpw6r8O0r/Doku--Select-%E2%9C%85?node-id=3-1110&embed-host=share"
		description="Use short, distinct option labels and a logical ordering (alphabetical or by frequency)."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/HmUqDoWeEMtoZSpw6r8O0r/Doku--Select-%E2%9C%85?node-id=3-1112&embed-host=share"
		description="Put very long lists into a basic select without search or grouping support."
	/>
</div>

###### Always provide a visible label

Every Select needs a clear label describing what the choice controls. Do not rely on placeholder text as a label, because it can be misunderstood and may not be available to all users.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/HmUqDoWeEMtoZSpw6r8O0r/Doku--Select-%E2%9C%85?node-id=3-1113&embed-host=share"
		description="Provide a short, specific label aligned consistently with other form fields."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/HmUqDoWeEMtoZSpw6r8O0r/Doku--Select-%E2%9C%85?node-id=3-1114&embed-host=share"
		description="Omit the label or use placeholder-only labeling to explain the field."
	/>
</div>

###### Handle empty, default, and disabled states intentionally

Define whether a default is appropriate. If selection is required, make the empty state explicit (e.g., "Choose an option") and validate appropriately. Disable the control only when it cannot be used, and explain why if needed.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/HmUqDoWeEMtoZSpw6r8O0r/Doku--Select-%E2%9C%85?node-id=3-1116&embed-host=share"
		description="Use a clear empty option/placeholder and validate when required; disable only with a clear reason."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/HmUqDoWeEMtoZSpw6r8O0r/Doku--Select-%E2%9C%85?node-id=3-1117&embed-host=share"
		description="Preselect a risky default that implies consent or hides that the user must choose."
	/>
</div>

###### Width and alignment must follow the layout grid

Select should align with surrounding inputs and respect container widths. Avoid arbitrary widths that break form rhythm; keep the dropdown aligned to the field.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/HmUqDoWeEMtoZSpw6r8O0r/Doku--Select-%E2%9C%85?node-id=3-1118&embed-host=share"
		description="Match Select width to the form layout and keep alignment consistent across the page."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/HmUqDoWeEMtoZSpw6r8O0r/Doku--Select-%E2%9C%85?node-id=3-1119&embed-host=share"
		description="Use inconsistent widths that misalign labels/fields or cause uneven spacing in forms."
	/>
</div>

###### Helper text, validation, and accessibility

Use helper text to prevent mistakes (format, constraints). Use validation to explain what's wrong and how to fix it. Ensure the Select has an accessible name; if the visible label is hidden, provide screenreader label text and expose required/invalid state.
