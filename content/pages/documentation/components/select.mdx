Select lets users choose one option from a predefined list via a dropdown. It is best when options are mutually exclusive and users don't need to see all choices at once, while still providing clear labeling, guidance, and validation.

- Choose one value from a known set (e.g., country, category, priority)
- Save space compared to showing all options as radios
- Provide a controlled input when free text is not allowed

## Usage Hints

<CollapsibleWrapper collapsedHeight="320px">

###### Select vs Radio

The select component should be used when the number of choices exceeds what can be reasonably displayed on screen without cluttering the UI. Typically, this applies when there are more than five options. Radio buttons are better suited for selecting from fewer options, as all options are visible at once.

###### Select vs Custom Select

Prefer select for simple single-choice lists. It uses the operating system's native dropdown and should be preferred if it is suitable for the use case. Use custom select when you need multi-select, search, grouping, or a clearer representation of multiple selected values.

	/>
	<GuidelineExample

	/>
</div>

###### Labels

Labels must be clear and concise. Do not rely on placeholder text as a label, because it disappears when a selection has been made.

<div className="guideline-examples">

###### Width

Select fields in responsive layouts should not take up the entire width of a wide screen (tablet and desktop). They should reflect a realistic length of content and be based on the columns or layout grid.

###### Message

Use the message to prevent mistakes. Use validation to explain what's wrong and how to fix it. Keep the message short and concise.
