## Guidelines

<CollapsibleWrapper collapsedHeight="320px">

### Responsive Differences

The mobile Shell can differ from its desktop counterpart. Adapt navigation structure to mobile constraints rather than mirroring desktop 1:1.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/RcpbbqfJwjNRlOwPhAX1im/Doku--Shell?node-id=3-1118&embed-host=share"
		description="Simplify navigation for mobile by reducing depth and grouping content differently if needed."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/RcpbbqfJwjNRlOwPhAX1im/Doku--Shell?node-id=3-1119&embed-host=share"
		description="Do not mirror the full desktop navigation hierarchy on mobile when it results in too many levels or small touch targets."
	/>
</div>

### FlatIcon Navigation

FlatIcon (bottom tab bar) works best for apps with a small set of equally important top-level destinations. It provides instant access without opening a menu but has limited capacity.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/RcpbbqfJwjNRlOwPhAX1im/Doku--Shell?node-id=3-1122&embed-host=share"
		description="Use FlatIcon when the application has 2–6 equally weighted top-level sections that users switch between frequently."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/RcpbbqfJwjNRlOwPhAX1im/Doku--Shell?node-id=3-1123&embed-host=share"
		description="Do not place primary or secondary actions in the FlatIcon bar. It is reserved for navigation destinations only."
	/>
</div>

### SubNavigation (Single Level)

The mobile SubNavigation supports only one navigation level. Keep the structure flat and the number of items manageable for horizontal scrolling.

<div className="guideline-examples">
	<GuidelineExample
		variant="do"
		figmaUrl="https://embed.figma.com/design/RcpbbqfJwjNRlOwPhAX1im/Doku--Shell?node-id=4049-3&embed-host=share"
		description="Use a single level of navigation items in the mobile SubNavigation with clear, short labels."
	/>
	<GuidelineExample
		variant="dont"
		figmaUrl="https://embed.figma.com/design/RcpbbqfJwjNRlOwPhAX1im/Doku--Shell?node-id=4049-4&embed-host=share"
		description="Do not overload the horizontal SubNavigation with too many items. Only one level is supported and excessive scrolling reduces discoverability."
	/>
</div>

### ControlPanel Position

Select the ControlPanel position based on the app type. Top is the default. Bottom supports thumb-reachable navigation for app-like experiences. Both use a Drawer for full navigation access.

<div className="guideline-examples">
	<GuidelineExample
