## Implementierung

Es gibt keine eigenständige Backdrop-Komponente. Wenn ein Backdrop wie im DBDrawer benötigt wird, können folgende Styles angewendet werden:

<CollapsibleWrapper collapsedHeight="320px">

### CSS

```css
.my-backdrop {
	background-color: color(
		from var(--db-adaptive-on-bg-basic-emphasis-100-default) srgb r g b / var(--db-opacity-lg)
	);
}

dialog::backdrop {
	background-color: color(
		from var(--db-adaptive-on-bg-basic-emphasis-100-default) srgb r g b / var(--db-opacity-lg)
	);
}
```

### SCSS

```scss
@use '@db-ux/core-foundations/build/styles/colors';
@use '@db-ux/core-foundations/build/styles/variables';

.my-backdrop {
	background-color: color(
		from #{colors.$db-adaptive-on-bg-basic-emphasis-100-default} srgb r g b /
			#{variables.$db-opacity-lg}
	);
}

/* OR */

dialog::backdrop {
	background-color: color(
		from #{colors.$db-adaptive-on-bg-basic-emphasis-100-default} srgb r g b /
			#{variables.$db-opacity-lg}
	);
}
```

**Hinweis:** Für die schwache Variante die Variable `db-opacity-sm` verwenden.

## Richtlinien

### Aufmerksamkeit lenken

Backdrop verwenden, wenn Nutzer:innen sich auf eine einzelne aktive UI-Ebene konzentrieren und die Aufgabe im Overlay abschließen müssen, um auf der Hauptseite fortzufahren. Wenn Nutzer:innen weiterhin mit dem Hintergrund arbeiten müssen, kein Backdrop verwenden und stattdessen ein nicht-blockierendes Muster einsetzen.

### Kontext

Backdrop soll Ablenkung reduzieren, nicht notwendigen Kontext entfernen. Wenn Nutzer:innen Hintergrundinformationen benötigen, eine Alternative in Betracht ziehen (Seitenpanel, Inline-Erweiterung oder nicht-modales Popover).

### Barrierefreiheit

Sicherstellen, dass Vordergrundinhalte ausreichend Kontrast zum abgedunkelten Hintergrund haben. Wenn der Backdrop einen modalen Zustand anzeigt, den Fokus innerhalb des aktiven Overlays halten und verhindern, dass assistive Technologien Hintergrundinhalte erreichen.
