## Rework Foundation

### Typografie (Typo Tokens / Styles)

- **Einheitliche Schrift für Code und Design:** Ersetzung der bisherigen Schriften (DB Screen Sans im Code / DB Screen Flex in Figma) durch **DB Neo Screen Sans** und **DB Neo Head** in den bereitgestellten Typo Tokens.
- **Italic ergänzt:** Einführung eines _italic_ Styles für den Body Token.
- ‼️ Hinweis: Die **DB Neo Screen Flex** Schrift wird **bis Ende 2025 aus Figma entfernt**. Ersetze sie rechtzeitig, um zukünftige Probleme zu vermeiden.

<img
	src="/assets/migration-guides/v2/typo.png"
	alt="Typografie-Übersicht"
	style={{ maxWidth: '100%' }}
/>

### Color Variables

- **Neue Farbklasse:** Einführung von **vibrant** für eine lebendigere Farbverwendung.

<img
	src="/assets/migration-guides/v2/colors-vibrant.png"
	alt="Vibrant-Farben"
	style={{ maxWidth: '100%' }}
/>

- **Optimierte dunkle Farbstufen:** Verbesserungen bei dunkleren Abstufungen für klarere Gradationen.
- **Dark Mode optimiert:** Verbesserte Konsistenz und Harmonie im Vergleich zum Light Mode.

<img
	src="/assets/migration-guides/v2/colors-steps-backgrounds.png"
	alt="Farbstufen und Hintergründe"
	style={{ maxWidth: '100%' }}
/>

- **Interactive Tokens:** Überarbeitet für deutlichere Unterschiede zwischen **default**, **hovered** und **pressed** Zuständen.

<img
	src="/assets/migration-guides/v2/colors-interactive-colors.png"
	alt="Interaktive Farben"
	style={{ maxWidth: '100%' }}
/>

- **Entfernung der On-Origin-Farben:** Die interaktiven Farben ❌ **on-origin-hovered** und ❌ **on-origin-pressed** werden entfernt, da sie keine ausreichende Barrierefreiheit im gesamten System gewährleisten können.

## Komponenten: Bugfixes (Figma)

Die folgenden Fixes erfordern besondere Aufmerksamkeit. Eine Übersicht aller weiteren behobenen Bugs findest du im **Issue-Teaser** am Ende der Seite.

### Badge

- **Emphasis Strong-Optimierung:** Verwendung von **vibrant** anstelle von **contrast-high** als Hintergrundfarbe für ein lebendigeres Badge-Erscheinungsbild.

<img
	src="/assets/migration-guides/v2/migration-badge-colors.png"
	alt="Badge-Farben"
	style={{ maxWidth: '100%' }}
/>

### Tag

- **Entfernung von Emphasis Origin:** Diese Variante wird entfernt.
- **Emphasis Strong-Optimierung:** Verwendung von **vibrant** anstelle von **contrast-high** als Hintergrundfarbe für ein lebendigeres Tag-Erscheinungsbild.
- **Übergangsphase für Design:**
  - Entfernte Varianten werden zunächst als **deprecated** ❌ markiert, um einen schrittweisen Ersatz zu ermöglichen.
- **Löschung mit v2.1.0:** Die vollständige Entfernung der Origin-Emphasis erfolgt mit dem nächsten Minor Release.

<img
	src="/assets/migration-guides/v2/migration-tag-colors.png"
	alt="Tag-Farben"
	style={{ maxWidth: '100%' }}
/>

### Formularfelder (Input, Select, Textarea)

- **Einheitliche Mindestbreite:** Konsistente Mindestbreiten für alle **Dateneingabe-Komponenten**.
- Input-Spezifikationen:
  - Ohne Icon: **db-sizing-lg**
  - 1 Icon: **db-sizing-xl**
  - 2 Icons: **db-sizing-2xl**
- Select-Spezifikationen:
  - Ohne Icon: **db-sizing-xl**
  - 1 Icon: **db-sizing-2xl**
- Textarea: **db-sizing-lg**
