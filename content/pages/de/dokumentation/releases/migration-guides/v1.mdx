## Komponenten: Property Alignment

### Allgemein

Das mit v0.6.0 begonnene Property Alignment zwischen Design und Code wurde für weitere Komponenten fortgeführt.

- **Betroffene Komponenten:**
  - Accordion
  - Card
  - Divider
  - Navigation
  - Popover
  - Section
  - Switch
  - Tag
  - Tooltip

- **Konsistenz und neue Properties:**
  Alle Subkomponenten innerhalb einer übergeordneten Komponente wurden vereinheitlicht.

- **Neue Development Properties (💻)**
  Properties, die für die Entwicklung eingeführt wurden, sind mit 💻 markiert (z. B. Prefix, Component, No Text, Children, Placement).

- **Design-spezifische Properties (🎨)**
  Properties, die nur für das Design relevant sind, sind mit 🎨 markiert.

- **Neue Varianten und Booleans:**
  - **Disabled Property (true/false):** Als dedizierte Variante eingeführt, Interaction State = None
  - **Readonly Property (true/false):** Als dedizierte Variante eingeführt, Interaction State = None
  - **true/false** für verschiedene Properties, die einen Zustand anzeigen

- **Verbesserte Barrierefreiheit:**
  - **Text Property für Screenreader:** Komponenten ohne sichtbaren Text (z. B. Icon Button, Icon Tag, Badge ohne Text) enthalten nun eine versteckte Text-Property für Screenreader.

- **Kleinere Anpassungen bei Property-Namen**, z. B.:
  - „Width: Auto Width/Full Width" → „Width: Auto/Full" (z. B. Button)
  - „🎨 xxx" → „Show xxx" (verschiedene Komponenten)

</DBSection>

### Komponenten

- **Popover**
  - Neben Auto Width gibt es nun eine Variante für Fixed Width-Verhalten mit einer maximalen Breite
  - Ein zusätzliches Spacing wurde ergänzt (lg)

- **Section**
  - Bestehende Breiten wurden in T-Shirt-Größen umbenannt und durch eine Small-Variante ergänzt
  - Size wurde in Spacing umbenannt und durch eine None-Variante ergänzt

- **Tag**
  - Die zusätzliche **Emphasis „Origin" wurde hinzugefügt.**
    Sie ermöglicht eine barrierefreie Verwendung des Tags mit einer lebendigeren Farbstufe aus der Farbpalette.

    <img

    	style={{ maxWidth: '100%' }}
    />

  - Die **Farben und Farblogik** über die verschiedenen Tag-Varianten wurden **angepasst und vereinheitlicht.** Vielen Dank für euer Feedback dazu! (siehe Rework Foundation → Color Variables)

    <img
    	src="/assets/migration-guides/v1/migration-tag-colors.png"
    	alt="Tag-Farben"
    	style={{ maxWidth: '100%' }}
    />

  - Ein **Slot wurde im Inhaltsbereich** des Tags hinzugefügt, um eine flexiblere Integration von eigenen Inhalten zu ermöglichen. **Bitte beachte die Dokumentation zur korrekten Verwendung!**

    <img
    	src="/assets/migration-guides/v1/migration-tag-slot.png"
    	alt="Tag Slot-Inhalt"
    	style={{ maxWidth: '100%' }}
    />

  - Die **Interactive-Variante** des Tags wurde **strukturell und inhaltlich überarbeitet:** - **Aufteilung des Interactive Tags in zwei Varianten** basierend auf dem Verhalten: - **Interactive Toggle** (ehemals Interactive): Diese Variante bildet das Verhalten des Tags als Checkbox oder Radio ab und zeigt die Zustände „Not selected" (checked = false) und „Selected" (checked = true) - **Interactive** (neu): Diese Variante bildet das Verhalten des Tags als Button oder Link ab

            - Für **Interactive Toggle** wurden die beiden Zustände „Not selected" (checked = false) und „Selected" (checked = true) für eine bessere Usability der Komponente zusammengeführt.

            - Ein (optionales) **Checked Status Icon wurde hinzugefügt**. Es ermöglicht eine bessere Unterscheidung zwischen dem Interactive Toggle Tag und dem Static Tag bei paralleler Verwendung innerhalb einer Anwendung.

## Rework Foundations

### Variable Modes (Figma)

- Entfernung veralteter „Density"-Modi wie ❌ **Tablet-XXX** und ❌ **Mobile-XXX.**
- Umbenennung von „Display Mode" zu „Mode"

### Color Variables

- Die **Auswahl der Markenfarben als Origin-Farben** wurde überarbeitet und harmonisiert, indem nicht nur Level 500, sondern verschiedene passende Abstufungen gewählt wurden. Alle Origin-Farben wurden angepasst, um **Barrierefreiheitsanforderungen und Komponentenverwendung** zu erfüllen und eine **konsistente Farbdarstellung von Origin- und On-Origin-Farben** sicherzustellen, insbesondere durch die Kombination dunkler Farben auf hellen Hintergründen. Dies erreicht eine **konsistente Darstellung** für alle semantischen und zusätzlichen Farben und **verbessert die Unterscheidbarkeit von Brand Red.**

## Umstrukturierung der Icon Libraries

- **Zusammenführung der bestehenden funktionalen Icon Libraries** „Core Icons" und „Extension PV Icons" in eine einzige Datei für Design und Entwicklung
  → **DB Theme Icons** inklusive aller Core Icons und PV Icons

<img
	src="/assets/migration-guides/v1/migration-icons.png"
	alt="Zusammenführung der Icon Libraries"
	style={{ maxWidth: '100%' }}
/>

- ❗ **Hinweis für Figma Libraries:** Es gibt Icons in der PV Extension Library, die Duplikate sind und bereits unter anderen Namen in der Core Library existieren (z. B. Ticket/Passenger → Human/Person, Transport/Walking → Transport/Pedestrian). Diese sind nun als deprecated markiert und müssen manuell durch das entsprechende Icon aus der neuen Library ersetzt werden. Alle markierten Icons sollten unter ihrem alten Namen in der zusammengeführten Datei noch auffindbar sein. Nicht enthaltene Libraries werden migriert und sollten verlinkt bleiben.

<img
	src="/assets/migration-guides/v1/migration-icons-deprecated.png"
	alt="Veraltete Icons"
	style={{ maxWidth: '100%' }}
/>

- Die folgenden Icons sind deprecated und müssen ersetzt werden:
  - **Facilities → Human**
    - Restricted Mobility Toilet, Shower, Shower Men, Shower Women, Sink, WC, WC Men, WC Sign, WC Women
  - **Features**
    - Medical → Medial/Medical Cross
    - No Smoking → Human/Cigarette Disabled
    - Person with Blind Cane, Person with Rollator → Accessibility/Person with Blind Cane, Accessibility/Person with Rollator
    - Power Socket → Technology/Power Socket
    - Restaurant → Food/Knife & Fork
    - Stroller → Human/Stroller
    - Support Dog → Accessibility/Support Dog
  - **Journey**
    - Journey Destination → Map/Map Pin
  - **Seat → Compartment**
    - Handicapped → Accessibility/Person with Wheelchair
    - Infant Compartment → Human/Toys
  - **Station → Buildings**
    - Train Station, Market
  - **Sustainability → Environment**
    - Environmental Mobility Check → Environmental/Leaf
  - **Ticket:**
    - Best Price → Social/Crown
    - Passenger, Passengers → Human/Person, Persons
    - QR Code, QR Code Scan → Technology/QR Code, QR Code Scan
    - Travel Insurance → Security/Shield Check
  - **Transportation → Transportation** (Ausnahmen: S-Bahn, Subway)
    - Airplane

    - Bicycle Trailer → Bike Trailer
    - Car

    - Scooter
