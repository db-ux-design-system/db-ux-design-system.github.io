## Wie funktioniert Code Connect?

Code Connect ist ein Figma-Feature, das Design-Komponenten direkt mit ihren Code-Implementierungen verknüpft. Entwickler:innen sehen das exakte Code-Snippet für eine ausgewählte Komponentenkonfiguration direkt in Figmas Dev Mode.

Im DB UX Design System steuern die Komponenteneigenschaften in Figma die Code-Connect-Ausgabe. Es gibt zwei Kategorien:

1. **Design-Eigenschaften:** beeinflussen das visuelle Erscheinungsbild und sind immer auf der ersten Ebene sichtbar (z. B. Size, Label, Disabled).
2. **⚙️ Code-Connect-Eigenschaften:** nur für die Code-Generierung relevant, nicht für das visuelle Design. Wenn konfigurierbar, erscheinen sie auf der ersten Ebene mit einem ⚙️-Icon. Wenn nicht konfigurierbar, werden sie im Hintergrund verarbeitet.

## Für Designer:innen

Als Designer:in arbeitest du mit beiden Arten von Eigenschaften, wenn du eine Komponenteninstanz konfigurierst.

### Design-relevante Eigenschaften

Das sind die Eigenschaften, die du direkt siehst, wenn du eine Komponenteninstanz in Figma auswählst. Sie beeinflussen das visuelle Erscheinungsbild der Komponente.

Du konfigurierst diese wie gewohnt. Sie beeinflussen sowohl das Design als auch die generierte Code-Ausgabe.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>⚙️ Code-Connect-Eigenschaften</h3>

Code-Connect-Eigenschaften sind mit einem **⚙️**-Icon gekennzeichnet und nur für die Code-Generierung relevant. Sie ändern nicht das visuelle Design.

Wenn eine Komponente konfigurierbare Code-Connect-Eigenschaften hat, erscheinen sie direkt auf der ersten Ebene am Ende der Eigenschaftsliste. Du erkennst sie am ⚙️-Präfix.

Allerdings haben nicht alle Komponenten konfigurierbare Code-Connect-Eigenschaften. Einige Komponenten (wie der Outline Button) enthalten intern einen **⚙️ Code Connect**-Layer, der „blinde" Eigenschaften an die Code-Ausgabe weitergibt (z. B. `variant="outlined"`). Da es nichts anzupassen gibt, werden keine ⚙️-Eigenschaften auf der ersten Ebene angezeigt.

## Für Entwickler:innen

{/* TODO: This section needs review by a developer familiar with the Code Connect implementation. */}

Als Entwickler:in bietet dir Code Connect direkten Zugriff auf das korrekte Code-Snippet für jede Komponentenkonfiguration, direkt in Figmas Dev Mode.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>So funktioniert es</h3>

Code Connect bildet Figma-Komponenteneigenschaften auf framework-spezifische Code-Snippets ab. Wenn Designer:innen eine Komponente konfigurieren (sowohl design-relevante als auch Code-Connect-Eigenschaften), zeigt das Dev-Mode-Panel das entsprechende Code-Snippet, das exakt dieser Konfiguration entspricht.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>Code Connect im Dev Mode verwenden</h3>

1. Öffne eine Figma-Datei und wechsle in den **Dev Mode** (Umschalter in der unteren Toolbar).
2. Wähle eine Komponenteninstanz auf der Canvas aus.
3. Im **Inspect-Panel** auf der rechten Seite findest du den **Code Connect**-Bereich (markiert als „Connected").
4. Wähle dein Ziel-Framework (z. B. React) aus dem Dropdown.
5. Das generierte Code-Snippet spiegelt die aktuelle Eigenschaftskonfiguration wider, einschließlich der über ⚙️ Code-Connect-Eigenschaften gesetzten Werte.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>Eigenschafts-Mapping</h3>

<div style={{ marginTop: 'var(--db-spacing-fixed-sm)' }}>

| Eigenschaftstyp          | Konfiguriert durch               | Beispiel-Eigenschaften                 | Im Code widergespiegelt |
| ------------------------ | -------------------------------- | -------------------------------------- | ----------------------- |
| Design-relevant          | Designer:in (erste Komponentenebene) | `size`, `label`, `disabled`       | ✅ Ja                   |
| Nur Code (konfigurierbar) | Designer:in (⚙️ auf erster Ebene)   | `required`, `checked`, `indeterminate` | ✅ Ja            |
| Nur Code (automatisch)   | Durch Variante bestimmt          | `variant`, `noText`                    | ✅ Ja (automatisch)     |

</div>

Beide Eigenschaftstypen werden in der generierten Code-Ausgabe berücksichtigt. Die Unterscheidung ist nur für den Design-Workflow relevant. Im Code werden alle Eigenschaften gleich behandelt.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>Standardwerte</h3>

⚙️ Code-Connect-Eigenschaften haben sinnvolle Standardwerte vorkonfiguriert. Wenn Designer:innen einen Wert nicht explizit geändert haben, wird der Standardwert in der Code-Ausgabe verwendet. Das bedeutet, dass du dich darauf verlassen kannst, dass das generierte Snippet für die ausgewählte Konfiguration vollständig und korrekt ist.

<h3 style={{ marginTop: 'var(--db-spacing-fixed-lg)' }}>Eigene Komponenten verbinden</h3>

Folge den Anweisungen von <a href="https://developers.figma.com/docs/code-connect/quickstart-guide/" target="_blank" referer="no-referer">Figma Code Connect</a>, um eigene Komponenten mit Code Connect zu verknüpfen. Du benötigst einen gültigen Figma-Token mit Zugriffsrechten auf deine Komponente.
Um DB-UX-Komponenten wiederzuverwenden, kannst du `instance.findInstance('DBButton').executeTemplate().example` nutzen, um den korrekten Code einzubinden.
Der „Code", der in Figma angezeigt wird, ist nur ein String – du kannst ihn beliebig anpassen. Wir generieren alles innerhalb der CI/CD für alle Frameworks basierend auf <a href="https://github.com/db-ux-design-system/core-web/blob/main/packages/components/configs/plugins/figma/index.cjs" target="_blank" referer="no-referer">diesem</a> Skript.
Du kannst eine KI und diesen String verwenden, um deine eigene Logik anzupassen.
